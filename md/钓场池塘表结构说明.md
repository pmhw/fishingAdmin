# 钓场下池塘相关表结构说明

## 关联关系

```
fishing_venue（钓场）
    └── fishing_pond（池塘） venue_id -> fishing_venue.id
            ├── pond_region（钓位区域）     pond_id -> fishing_pond.id
            ├── pond_seat（钓位，独立+code） pond_id -> fishing_pond.id, region_id -> pond_region.id
            ├── pond_fee_rule（收费规则）   pond_id -> fishing_pond.id
            └── pond_return_rule（回鱼规则） pond_id -> fishing_pond.id
```

- 池塘必须归属某个钓场（`venue_id`）。
- 钓位区域、收费规则、回鱼规则均归属某个池塘（`pond_id`）。
- 删除钓场会级联删除其下池塘；删除池塘会级联删除其下区域/收费/回鱼规则。

## 2. 钓位区域表 `pond_region`

用于「按区域 + 序号范围」DIY 配置钓位，例如：西岸 1~29、中间浮桥 30~89。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 ID |
| name | VARCHAR(50) | 区域名称，如 西岸、中间浮桥 |
| start_no | INT UNSIGNED | 钓位起始序号（含） |
| end_no | INT UNSIGNED | 钓位结束序号（含） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

- 一条记录表示一个区域的钓位序号范围。
- 例如：西岸 1~29 → name=西岸, start_no=1, end_no=29；中间浮桥 30~89 → name=中间浮桥, start_no=30, end_no=89。
- 后台可自由添加/编辑区域及起止序号，实现用户 DIY 配置。
- **说明**：本表只描述「区域 + 序号范围」，没有为每个钓位号建单独一行，因此不能直接用于「用户钓鱼时临时关联座位号」做点餐、计费、回鱼。若需要按座位号关联，请使用下面的 **钓位表 pond_seat**（每个钓位一行，带唯一 code）。

---

## 2.1 钓位表 `pond_seat`（独立钓位 + 唯一 code，用于点餐/计费/回鱼）

**建表 SQL**：见 `md/pond_seat.sql`。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 |
| region_id | INT UNSIGNED | 所属区域（可选，对应 pond_region.id，便于按区域展示） |
| seat_no | INT UNSIGNED | 钓位序号，同池塘内唯一（1、2、…、89） |
| code | VARCHAR(32) | **唯一业务编码**，用于点餐/计费/回鱼时扫码或输入，如 A-01、P1-012 |
| status | VARCHAR(20) | 状态：idle-空闲，in_use-使用中（可选，用于选座） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

- **与 pond_region 的关系**：pond_region 仍是「区域 + 序号范围」的配置；pond_seat 是「每个钓位一行」的实体表。创建钓位时可从区域范围批量生成（如西岸 1~29 生成 29 条 pond_seat，并生成唯一 code，如 `XA-01`～`XA-29`）。
- **code 用途**：用户钓鱼时临时关联到某钓位后，点餐、计费、回鱼接口用 **code**（或 seat_id）定位，便于扫码、输入座位号等。
- **后续扩展**：可增加「开杆/订单/回鱼」等业务表，通过 `seat_id` 或 `seat_code` 关联到本表。

## 3. 收费规则表 `pond_fee_rule`

| 字段 | 类型 | 说明 |

---

## 4. 回鱼规则表 `pond_return_rule`

上下限为**条数范围**，回鱼方式为按斤或按条。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 ID |
| name | VARCHAR(100) | 规则名称，如 鲫鱼回鱼 |
| lower_limit | INT UNSIGNED | 下限（条数） |
| upper_limit | INT UNSIGNED | 上限（条数），0 可表示不限 |
| return_type | VARCHAR(10) | 回鱼方式：jin-按斤，tiao-按条 |
| amount | DECIMAL(10,2) | 金额（元/斤 或 元/条） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

---

## 执行顺序

1. 确保已存在 `fishing_venue` 表。
2. 执行 `md/pond_tables.sql` 创建池塘、区域、收费、回鱼四张表（含外键）。
3. 需要「独立钓位 + code」时，执行 `md/pond_seat.sql` 创建钓位表。
4. 若不需要外键约束，可去掉 SQL 中的 `CONSTRAINT ... FOREIGN KEY` 部分再执行。
