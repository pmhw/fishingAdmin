# 钓场下池塘相关表结构说明

## 关联关系

```
fishing_venue（钓场）
    └── fishing_pond（池塘） venue_id -> fishing_venue.id
            ├── pond_region（钓位区域）     pond_id -> fishing_pond.id
            ├── pond_fee_rule（收费规则）   pond_id -> fishing_pond.id
            └── pond_return_rule（回鱼规则） pond_id -> fishing_pond.id
```

- 池塘必须归属某个钓场（`venue_id`）。
- 钓位区域、收费规则、回鱼规则均归属某个池塘（`pond_id`）。
- 删除钓场会级联删除其下池塘；删除池塘会级联删除其下区域/收费/回鱼规则。

---

## 1. 池塘表 `fishing_pond`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| venue_id | INT UNSIGNED | 所属钓场 ID |
| name | VARCHAR(100) | 池塘名称 |
| images | TEXT | 池塘图片，JSON 数组 |
| pond_type | VARCHAR(20) | 池塘类型：black_pit-黑坑，jin_tang-斤塘，practice-练杆塘 |
| seat_count | INT UNSIGNED | 钓位数（个） |
| area_mu | DECIMAL(6,2) | 池塘面积（亩） |
| water_depth | VARCHAR(50) | 池塘水深，如 1.5-2米 |
| fish_species | VARCHAR(500) | 鱼种类型，逗号分隔多标签 |
| rod_rule | VARCHAR(500) | 限杆规则 |
| bait_rule | VARCHAR(500) | 限饵规则 |
| extra_rule | VARCHAR(500) | 补充规则 |
| open_time | DATE | 开塘时间 |
| status | VARCHAR(20) | 池塘状态：open-开放，closed-关闭 |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

---

## 2. 钓位区域表 `pond_region`

用于「按区域 + 序号范围」DIY 配置钓位，例如：西岸 1~29、中间浮桥 30~89。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 ID |
| name | VARCHAR(50) | 区域名称，如 西岸、中间浮桥 |
| start_no | INT UNSIGNED | 钓位起始序号（含） |
| end_no | INT UNSIGNED | 钓位结束序号（含） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

- 一条记录表示一个区域的钓位序号范围。
- 例如：西岸 1~29 → name=西岸, start_no=1, end_no=29；中间浮桥 30~89 → name=中间浮桥, start_no=30, end_no=89。
- 后台可自由添加/编辑区域及起止序号，实现用户 DIY 配置。

---

## 3. 收费规则表 `pond_fee_rule`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 ID |
| name | VARCHAR(100) | 收费名称，如 正钓4小时 |
| duration | VARCHAR(50) | 垂钓时长，如 4小时、1天 |
| amount | DECIMAL(10,2) | 收费金额（元） |
| deposit | DECIMAL(10,2) | 押金（元） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

---

## 4. 回鱼规则表 `pond_return_rule`

上下限为**条数范围**，回鱼方式为按斤或按条。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT UNSIGNED | 主键 |
| pond_id | INT UNSIGNED | 所属池塘 ID |
| name | VARCHAR(100) | 规则名称，如 鲫鱼回鱼 |
| lower_limit | INT UNSIGNED | 下限（条数） |
| upper_limit | INT UNSIGNED | 上限（条数），0 可表示不限 |
| return_type | VARCHAR(10) | 回鱼方式：jin-按斤，tiao-按条 |
| amount | DECIMAL(10,2) | 金额（元/斤 或 元/条） |
| sort_order | INT | 排序 |
| created_at / updated_at | DATETIME | 创建/更新时间 |

---

## 执行顺序

1. 确保已存在 `fishing_venue` 表。
2. 执行 `md/pond_tables.sql` 创建上述四张表（含外键）。
3. 若不需要外键约束，可去掉 SQL 中的 `CONSTRAINT ... FOREIGN KEY` 部分再执行。
