# 后台 · 新订单弹窗与声音提醒

## 实现方式（当前项目）

采用 **前端轮询**，无需 WebSocket / 额外中间件：

1. 接口：`GET /api/admin/trade/order-alert-tip`（需登录 + `admin.trade.order.manage`）
2. 返回当前管理员**可见钓场范围内**的：
   - `fishing_order_max_id`（开卡/预付类 `fishing_order` 表最大主键）
   - `shop_order_max_id`（店铺 `venue_shop_order` 表最大主键）
3. 后台顶栏 **「新单提醒」**：总开关、**弹窗**、**声音**；约 **每 25 秒** 请求一次。
4. 首次成功拉取仅作 **基线**，不提示；之后若任一侧 `max_id` 变大 → **Element Plus 通知** + **Web Audio 双音提示**（不依赖 mp3 文件）。

与 **角色钓场范围**（`admin_role_venue`）一致：仅统计这些钓场下的订单；未配置范围则视为全部钓场。

## 多门店场景

- **门店多 ≠ 请求成倍增加**：每个登录后台的管理员，无论负责 1 家店还是 100 家店，**每次轮询仍是 1 次 HTTP 请求**；后端用 `WHERE venue_id IN (...)` 在两条表上各算一次 `MAX(id)`，有索引时开销很小。
- **压力主要看「同时有多少人开着后台」**，不是门店总数。
- 已做优化：**浏览器标签页隐藏或最小化时自动暂停轮询**，切回前台再恢复（减少无效请求）。
- 若以后有 **几十人同时盯后台**，可把间隔再拉长，或上 WebSocket / Redis 计数（见上文「秒级」）。

## 使用说明

- 打开顶栏 **新单提醒** 开关即可；可单独关闭弹窗或声音。
- **浏览器策略**：部分浏览器要求页面内有过 **用户点击** 后才允许播放声音；若无声，在后台任意处点一下再试。
- 点击通知（若浏览器支持 `onClick`）会跳转订单列表（以店铺单优先逻辑，双新单时进店铺订单页）。
- 退出登录会停止轮询。

## 若以后要「秒级」提醒

可升级为：

- **WebSocket**（ThinkPHP + Swoole / Workerman，或独立 Node 推送服务）
- **SSE（Server-Sent Events）** 单向推送
- 微信 **模板消息 / 订阅消息** 给管理员手机提醒

轮询适合中小流量、实现成本最低。
