# 小程序 · 钓场店铺商品接口

基础路径：`/api/mini`。  
**分类 / 列表 / 详情** 无需登录（与钓场列表/详情一致）。  
**创建订单** 需登录（`Authorization: Bearer`）。  

仅展示 **`fishing_venue.status = 1`** 的钓场，且本店商品 **`venue_product.status = 1`**、公共商品 **`product.status = 1`**。

---

## 1. 本店分类列表

`GET /api/mini/venues/{venue_id}/shop/categories`

**响应 `data`：**

```json
{
  "list": [
    { "id": 1, "name": "饵料", "sort_order": 0 }
  ]
}
```

仅返回 **`venue_shop_category.status = 1`** 的分类。

---

## 2. 本店商品列表

`GET /api/mini/venues/{venue_id}/shop/products`

| Query | 说明 |
|-------|------|
| `page` | 页码，默认 1 |
| `limit` | 每页条数，默认 10，最大 50 |
| `shop_category_id` | 可选。**不传**=全部分组；**`0`**=仅未分组（`shop_category_id` 为空或 0）；**正整数**=该分类下商品 |

**列表项字段：**

| 字段 | 说明 |
|------|------|
| `venue_product_id` | 本店商品主键，**详情接口路径用此 id** |
| `product_id` | 公共库 SPU id |
| `shop_category_id` | 本店分类 id，可 null |
| `shop_category` | `{ id, name }` 或 null |
| `name`, `intro`, `cover_image`, `unit` | 来自公共商品 |
| `min_price` | 当前可售规格中的最低售价（元），无规格时为 null |
| `sku_count` | 可售规格数量（本店 SKU 与公共 SKU 均启用） |

---

## 3. 商品详情（含规格）

`GET /api/mini/venues/{venue_id}/shop/products/{vp_id}`

- **`vp_id`** = 列表返回的 **`venue_product_id`**

**响应 `data`：**

```json
{
  "venue_product_id": 12,
  "shop_category": { "id": 1, "name": "饵料" },
  "product": {
    "id": 3,
    "name": "xxx",
    "intro": "",
    "cover_image": "/storage/...",
    "images": [],
    "detail": "",
    "unit": "件"
  },
  "skus": [
    {
      "id": 101,
      "product_sku_id": 55,
      "spec_label": "500ml×1袋",
      "price": 12.5,
      "stock": 30
    }
  ],
  "min_price": 12.5
}
```

- `skus[].id` 为 **`venue_product_sku` 主键**，下单时使用该 id（见下节）。

- 仅返回 **`venue_product_sku.status = 1`** 且 **`product_sku.status = 1`** 的规格；`stock` 为当前库存（整数）。

## 4. 创建店铺订单（需登录）

`POST /api/mini/venues/{venue_id}/shop/orders`  

Header：`Authorization: Bearer {token}`  

**业务规则：** 用户须在本钓场有一条 **进行中的开钓单**（`fishing_session.status = ongoing` 且 `venue_id` 一致），否则返回 **400**，提示先开卡开钓后再买店铺商品。下单时会 **快照** 当前开钓单的池塘、钓位（`seat_no` / `seat_code`）写入店铺订单，供后台与支付页展示。

Body JSON：

```json
{
  "items": [
    { "venue_product_sku_id": 101, "quantity": 2 }
  ],
  "use_balance": true,
  "remark": "可选备注"
}
```

- `venue_product_sku_id`：详情里 `skus[].id`。同一 SKU 多行会自动合并数量。
- `use_balance`：可选，默认 `true`。仅 **`mini_user.is_vip = 1`** 时可用余额抵扣（与开钓单规则一致）。
- 成功即 **扣减库存**（未支付取消需后续做释放库存接口）。
- 订单号 **`SO` 开头**，与微信支付 `out_trade_no` 一致。

**响应 `data`（节选）：**

| 字段 | 说明 |
|------|------|
| `order_no` | 业务单号 |
| `status` | `pending` 待微信付 / `paid` 已付（含纯余额） |
| `pay_channel` | `wx_mini` / `balance` / `mixed` |
| `goods_amount_yuan` | 商品合计（元） |
| `balance_deduct_yuan` | 余额抵扣（元） |
| `need_pay_yuan` / `need_pay_fen` | 还需微信支付部分 |
| `mini_pay_path` | 有微信待付时，可跳转支付页的路径（含 order_no、amount） |
| `fishing_session_id` | 关联的开钓单 id |
| `pond_id` / `seat_no` / `seat_code` | 下单时座位快照（可空） |

### 微信支付（复用现有接口）

若 `need_pay_fen > 0`：

1. `GET /api/mini/orders/{order_no}` 拉单（与开钓单支付页相同）。
2. `POST /api/mini/pay/wechat/jsapi`，Body 建议：`order_no`、`description`=`店铺订单`（金额以服务端 `fishing_order` 为准，勿篡改 `total_fee`）。
3. 支付成功后，微信回调会把 **`venue_shop_order`** 同步为 `paid`（与 `fishing_order` 一致）。

若余额已付清（`need_pay_fen === 0`），无 `fishing_order` 行，仅 `venue_shop_order.status = paid`，同样可用 `GET /api/mini/orders/{order_no}` 查询（返回 `order_kind: shop`）。

## 5. 小程序调用示例

```javascript
const base = 'https://你的域名/api/mini'
const venueId = 1

// 分类
wx.request({ url: `${base}/venues/${venueId}/shop/categories` })

// 商品列表（某分类）
wx.request({ url: `${base}/venues/${venueId}/shop/products`, data: { page: 1, limit: 10, shop_category_id: 2 } })

// 商品详情
const vpId = 12
wx.request({ url: `${base}/venues/${venueId}/shop/products/${vpId}` })

// 下单（需 token）
wx.request({
  url: `${base}/venues/${venueId}/shop/orders`,
  method: 'POST',
  header: { Authorization: 'Bearer ' + token, 'Content-Type': 'application/json' },
  data: { items: [{ venue_product_sku_id: 101, quantity: 1 }], use_balance: true }
})
```

## 6. 数据库依赖

需已执行：

- `md/venue_shop_product.sql`（及如有旧库则 `md/venue_shop_category_migration.sql`）
- **`md/venue_shop_order.sql`**（店铺订单主表 + 明细）
- **`md/venue_shop_order_session_seat.sql`**（订单上增加开钓单 id、池塘、座位快照；老库必执行）

存在 `venue_shop_category`、`venue_product`、`venue_product_sku`、`venue_shop_order`、`venue_shop_order_item` 等表；微信支付依赖已有 **`fishing_order`** 表。
