## 开钓单 / 开卡业务说明（后台 + 小程序）

### 1. 业务场景概述

- “开钓单”即“开卡”：为某个小程序用户，在指定钓场 / 池塘 / 钓位上开启一场钓鱼会话。
- 一次开钓单会关联：
  - 小程序用户（会员）
  - 钓场（`fishing_venue`）
  - 池塘（`fishing_pond`）
  - 钓位（`pond_seat`，可选）
  - 收费规则（`pond_fee_rule`，包含钓费 + 押金 + 时长）
  - 支付订单（`fishing_order`）
  - 开钓会话（`fishing_session`）

### 2. 金额构成与会员免押金规则

#### 2.1 收费规则拆分

后端从 `pond_fee_rule` 中取出：

- 钓费金额：`fee.amount`（单位：元）
- 押金金额：`fee.deposit`（单位：元）

换算为分后：

- `amountFen = fee.amount * 100`（钓费）
- `depositRawFen = fee.deposit * 100`（原始押金）

#### 2.2 会员免押金逻辑

当满足以下条件时，押金免收：

- 用户是会员：`mini_user.is_vip = 1`
- 后台开钓单时勾选“会员余额抵扣”：`use_balance = true`
- 收费规则中押金金额 `depositRawFen > 0`

此时：

- 实际押金：`depositTotalFen = 0`（免押金）
- 钓费仍按原金额收取
- 应收总额：`amountTotalFen = amountFen + depositTotalFen`

否则（非会员或未使用余额抵扣），押金照常收取：

- `depositTotalFen = depositRawFen`
- `amountTotalFen = amountFen + depositTotalFen`

#### 2.3 会员余额抵扣

若配置了 `mini_user.balance`（元），且：

- `user.is_vip = 1`
- 后台开钓单时 `use_balance = true`

则按如下规则进行余额抵扣：

- 余额（分）：`userBalanceFen = balance * 100`
- 抵扣金额：`balanceDeductFen = min(userBalanceFen, amountTotalFen)`
- 剩余需微信支付：`needPayFen = amountTotalFen - balanceDeductFen`
- 扣减用户余额：`user.balance -= balanceDeductFen / 100`

### 3. 订单与开钓单的生成逻辑

#### 3.1 有微信支付金额（需扫码支付）

条件：`needPayFen > 0`

流程：

1. 后台仅创建一条待支付订单 `fishing_order`：
   - `order_no`：业务订单号（如 `O202603170001xxxx`）
   - `amount_total = needPayFen`（仅记录需微信支付部分）
   - `amount_paid = 0`
   - `status = 'pending'`
   - `pay_channel = 'wx_mini'`
2. 生成小程序支付路径与二维码：
   - 支付页路径：`/pages/pay/index?order_no=...&amount=...`
   - 小程序码：调用 `getwxacodeunlimit`，`page = pages/pay/index`，`scene = "order_no=..."`（仅传订单号，长度 < 32）
3. **此时不会立即创建 `fishing_session`**，座位也不会占用。
4. 小程序端扫码 / 跳转到支付页，调起微信支付。
5. 微信支付回调 `PayController::notify()` 成功后：
   - 将 `fishing_order.status` 标记为 `paid`，记录 `amount_paid`、`pay_time` 等。
   - 调用公共逻辑 `createSessionAfterPaid($order)`：
     - 校验钓场 / 池塘 / 钓位 / 收费规则关系是否正确。
     - 确认该钓位当前没有进行中的开钓单（`status = 'ongoing'`）。
     - 按收费规则重新拆分应收金额、押金，计算应付金额与自动到期时间 `expire_time`。
     - 创建一条 `fishing_session`：
       - `status = 'ongoing'`
       - `order_id` 指向已支付订单
       - `seat_id / seat_no / seat_code` 等字段冗余保存
       - `amount_total` / `amount_paid` / `deposit_total` 等金额字段写入
     - 此时座位开始占用，后台开钓单列表能够看到记录。

#### 3.2 完全由余额支付（会员余额足额覆盖）

条件：`needPayFen == 0`

流程：

1. 后台创建一条“余额支付订单” `fishing_order`：
   - `description = '开钓单余额支付'`
   - `amount_total = amountTotalFen`（应收总额 = 钓费 + 实际押金；如会员免押金则仅钓费）
   - `amount_paid = amountTotalFen`（全部已由余额支付）
   - `status = 'paid'`
   - `pay_channel = 'balance'`
2. **立即创建开钓单 `fishing_session`**（无需等待微信回调）：
   - `status = 'ongoing'`
   - `order_id` 关联上述余额订单
   - `start_time = 当前时间`
   - `expire_time` 按收费规则时长计算
   - `amount_total = amountTotalFen`
   - `amount_paid = amountTotalFen`
   - `deposit_total = depositTotalFen`（会员免押金则为 0）
3. 座位立即占用，后台开钓单列表立即可见该条记录。

### 4. 后台开卡页面提示文案

后台“经营管理 - 开钓单”页面在手动开钓成功后，会弹出提示，内容包括：

- 余额抵扣金额：`balance_deduct`（元）
- 还需微信支付金额：`need_pay`（元）
- 费用拆分：
  - 钓费：`fee_amount`（元）
  - 原始押金：`fee_deposit`（元）
  - 实际收取押金：`deposit_effective`（元）
- 会员免押金提示（满足免押金条件时）：
  - “会员免押金，本次实际收取押金 0 元”

示例（会员免押金场景）：

- 收费规则：钓费 260 元，押金 100 元；
- 会员 + 使用余额抵扣，余额充足；

提示大致为：

> 开钓订单创建成功。余额抵扣：360 元，待支付：0 元  
> 钓费：260 元，押金：100 元（会员免押金，本次实际收取押金 0 元）

### 5. 小程序端支付页配合说明

1. 后台返回的支付路径统一为：
   - `/pages/pay/index?order_no=Oxxx&amount=68.00`
2. 扫码进入时，小程序通过 `scene` 解析出 `order_no`，再调用：
   - `GET /api/mini/orders/:order_no`（需带小程序登录 token）
3. 订单接口返回：
   - 金额字段：`amount_total_yuan`、`amount_paid_yuan`、`need_pay_yuan`、`amount`
   - 订单用户 openid：`mini_user_openid`
   - 当前登录用户是否为订单所有人标记：`openid_match`
4. 小程序支付页根据：
   - `amount` 展示金额
   - `openid_match` 决定是否显示“立即支付”按钮（扫码代付场景可隐藏）
5. 实际发起支付时，走已有的：
   - `POST /api/mini/pay/wechat/jsapi`
   - 调用 `wx.requestPayment` 完成支付。
