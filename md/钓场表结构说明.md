# 钓场表结构规划

## 主表：fishing_venue（钓场）

| 字段 | 类型 | 说明 |
|------|------|------|
| **基础信息** | | |
| id | INT UNSIGNED | 主键自增 |
| name | VARCHAR(100) | 钓场名称 |
| intro | VARCHAR(500) | 简短简介（列表/卡片展示） |
| description | TEXT | 详细描述（富文本或纯文本） |
| cover_image | VARCHAR(255) | 封面图 URL（单张） |
| images | TEXT / JSON | 多图（JSON 数组存储多张图路径，如 ["/storage/venue/1.jpg","/storage/venue/2.jpg"]） |
| **位置与联系** | | |
| province | VARCHAR(50) | 省 |
| city | VARCHAR(50) | 市 |
| district | VARCHAR(50) | 区/县 |
| address | VARCHAR(255) | 详细地址 |
| longitude | DECIMAL(10,7) | 经度（地图定位） |
| latitude | DECIMAL(10,7) | 纬度（地图定位） |
| contact_phone | VARCHAR(30) | 联系电话 |
| contact_wechat | VARCHAR(64) | 微信/客服（选填） |
| **营业与收费** | | |
| opening_hours | VARCHAR(200) | 营业时间（如 "08:00-18:00" 或 "全天"） |
| price_type | VARCHAR(20) | 计费方式：day-按天, jin-按斤, mix-混合, free-免费 |
| price_info | VARCHAR(255) | 价格说明（如 "80元/天"、"12元/斤"） |
| price_min | DECIMAL(10,2) | 最低价（元，用于筛选/排序） |
| price_max | DECIMAL(10,2) | 最高价（元，选填） |
| **扩展信息** | | |
| facilities | VARCHAR(500) | 设施（逗号分隔或 JSON，如 停车场,餐饮,厕所） |
| fish_species | VARCHAR(500) | 鱼种（逗号分隔或 JSON，如 鲫鱼,鲤鱼,草鱼） |
| **状态与排序** | | |
| status | TINYINT | 状态：1-上架 0-下架 2-待审核 |
| sort_order | INT | 排序值，越小越靠前 |
| view_count | INT UNSIGNED | 浏览量（可选） |
| **时间** | | |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

---

## 可选扩展

- **钓场-标签关联表**：若标签多、需单独管理，可做 `venue_tag` + `tag` 表。
- **钓场-鱼种关联表**：若鱼种要统一维护、多选，可做 `venue_fish` + `fish_species` 表。
- **收藏/点赞**：若小程序端要做收藏，可加 `user_favorite_venue`（user_id, venue_id）。

---

## 地图选点配置（后台全局配置）

钓场「地图选点」使用高德地图，Key 从**杂项 - 全局配置**读取，需在配置中心添加：

| 变量名 (key) | 说明 |
|--------------|------|
| amap_key | 高德开放平台 Web 端 (JSAPI) Key |
| amap_security_code | 高德安全密钥（JSAPI 2.0 必填） |

---

## 索引建议

- `idx_status`：按状态筛选上架列表。
- `idx_province_city`：按省/市筛选。
- `idx_sort_order`：排序。
- `idx_price_min`：按价格区间筛选（可选）。
