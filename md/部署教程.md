### 搭建线上

1. 导入代码到服务器

2. 配置站点目录（例如宝塔/面板）
   - 程序目录：根目录
   - 运行目录：`public`
   - 伪静态：`thinkphp8`

3. 安装依赖（在**包含 `composer.json` 的目录**执行，一般是项目根或子目录 `fishingAdmin`）

   - 若项目在子目录 `fishingAdmin`（运行目录为 `.../fishingAdmin/public`）
     ```sh
     cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin
     composer install --no-dev --optimize-autoloader
     ```

   - 若项目直接在站点根目录（即 `/www/wwwroot/fishing.kaiyuantong.cn` 下有 `composer.json`）
     ```sh
     cd /www/wwwroot/fishing.kaiyuantong.cn
     composer install --no-dev --optimize-autoloader
     ```

   **注意：** 不要用 root 跑 Composer。可用 `composer install ... --no-interaction` 避免交互，或用非 root 用户执行。

4. 修改 `.env` 内的数据库信息

5. 创建 `runtime` 目录并赋予写权限（否则登录等会报错：`Failed to open stream: No such file or directory`）
   ```sh
   cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin
   mkdir -p runtime/cache runtime/log runtime/session runtime/temp
   chmod -R 755 runtime
   # 若仍报错，可改为可写：chmod -R 777 runtime
   ```

6. 上传图片可访问：在 `public` 目录建立 storage 软链，使 `/storage/...` 能访问到 runtime 里的上传文件（否则轮播图等上传后预览 404）
   ```sh
   cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin/public
   ln -s ../runtime/storage storage
   ```

7. 后台管理员表与接口（可选）
   - 在库中执行 `md/admin_user.sql` 创建表；若表为空可请求 `POST /api/admin/init` 创建首个管理员，再通过 `POST /api/admin/login` 登录。

7.1 钓场店铺商品表（使用「公共商品库 / 钓场选品」必做）
   - 在**与 `.env` 相同的数据库**里执行 `md/venue_shop_product.sql`（开头注释里有 `USE \`库名\`;` 示例）。
   - 若报错 `Table 'xxx.product' doesn't exist`，说明该库未执行过此脚本或选错了库。
   - 脚本依赖已有表 `fishing_venue`；末尾会写入店铺相关后台权限，需 `super_admin` 等角色已有。
   - 若你**早已执行过旧版**脚本（无 `venue_shop_category`、无 `venue_product.shop_category_id`），请再执行 `md/venue_shop_category_migration.sql` 增量升级。

8. 小程序会员码（动态二维码）配置（可选，防截图失效）
   1) 在数据库 `system_config` 表新增以下配置：
      - `vip_code_secret`：签名密钥（不能为空）
        - 签名算法：HMAC-SHA256（后端会对 HMAC 做截断再拼到 scene 中）
        - 推荐长度：**至少 32 位字符**，建议 **64 位以上**
      - `vip_code_ttl_seconds`：有效期秒数（可选，默认 300 秒 = 5 分钟）

   2) 建议执行 SQL（把值替换成你自己的）：
      ```sql
      USE fishing_kaiyuant;

      INSERT INTO system_config (config_key, config_value, remark)
      VALUES
        ('vip_code_secret', '替换成你的随机密钥', '小程序会员码签名密钥'),
        ('vip_code_ttl_seconds', '300', '小程序会员码有效期（秒）')
      ON DUPLICATE KEY UPDATE
        config_value = VALUES(config_value),
        remark = VALUES(remark);
      ```

9. 后台前端（本地开发）：`admin/` 目录执行 `cd admin && npm i && npm run dev`，访问 `http://localhost:5174`

### 经营管理 - 开钓单自动结束（方案A：到期自动释放座位）

#### 1. 数据库迁移

若已创建 `fishing_session` 表但缺少 `expire_time` 字段，请执行：

```sql
ALTER TABLE `fishing_session`
  ADD COLUMN `expire_time` DATETIME DEFAULT NULL COMMENT '到期时间（start_time + 时长）' AFTER `start_time`,
  ADD KEY `idx_expire_time` (`expire_time`);
```

#### 2. 定时任务（推荐）

项目内已提供命令：`php think session:expire`  
作用：将所有 `status=ongoing` 且 `expire_time <= now()` 的开钓单自动改为 `finished`，并写入 `end_time`，从而释放座位占用。

建议在服务器增加 crontab，每分钟执行一次：

```sh
* * * * * cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin && php think session:expire >> /tmp/session_expire.log 2>&1
```

#### 3. 兜底机制（防止定时任务失效）

即使 crontab 临时失效，系统也会在用户/后台发起请求时做轻量到期检查（有节流），自动结束到期的开钓单，避免座位长期占用无法释放。
