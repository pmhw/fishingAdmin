### 搭建线上
1. 导入代码到服务器

2. 设置程序目录为根目录、设置运行目录为 public、设置伪静态为 thinkphp8

3. 安装依赖（在**有 composer.json 的目录**执行，一般是项目根或子目录 `fishingAdmin`）：

```sh
# 若项目在子目录 fishingAdmin（运行目录为 .../fishingAdmin/public）
cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin
composer install --no-dev --optimize-autoloader
```

若项目直接在站点根目录（即 /www/wwwroot/fishing.kaiyuantong.cn 下就有 composer.json），则：

```sh
cd /www/wwwroot/fishing.kaiyuantong.cn
composer install --no-dev --optimize-autoloader
```

**注意：** 不要用 root 跑 Composer。可用 `composer install ... --no-interaction` 避免交互，或用非 root 用户执行。


4. 修改 .env 内的数据库信息

5. **创建 runtime 目录并赋予写权限**（否则登录等会报错：Failed to open stream: No such file or directory）
```sh
cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin
mkdir -p runtime/cache runtime/log runtime/session runtime/temp
chmod -R 755 runtime
# 若仍报错，可改为可写：chmod -R 777 runtime
```

5.1 **上传图片可访问**：在 **public** 目录下建立 storage 软链，使 `/storage/...` 能访问到 runtime 里的上传文件（否则轮播图等上传后预览 404）：
```sh
cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin/public
ln -s ../runtime/storage storage
```

6. 后台管理员表与接口（可选）：在库中执行 `md/admin_user.sql` 创建表；若表为空可请求 `POST /api/admin/init` 创建首个管理员，再通过 `POST /api/admin/login` 登录。

7. 后台前端在 `admin/` 目录，本地：`cd admin && npm i && npm run dev`，访问 http://localhost:5174。


### 经营管理 - 开钓单自动结束（方案A：到期自动释放座位）

#### 1. 数据库迁移

若已创建 `fishing_session` 表但缺少 `expire_time` 字段，请执行：

```sql
ALTER TABLE `fishing_session`
  ADD COLUMN `expire_time` DATETIME DEFAULT NULL COMMENT '到期时间（start_time + 时长）' AFTER `start_time`,
  ADD KEY `idx_expire_time` (`expire_time`);
```

#### 2. 定时任务（推荐）

项目内已提供命令：`php think session:expire`  
作用：将所有 `status=ongoing` 且 `expire_time <= now()` 的开钓单自动改为 `finished`，并写入 `end_time`，从而释放座位占用。

建议在服务器增加 crontab，每分钟执行一次：

```sh
* * * * * cd /www/wwwroot/fishing.kaiyuantong.cn/fishingAdmin && php think session:expire >> /tmp/session_expire.log 2>&1
```

#### 3. 兜底机制（防止定时任务失效）

即使 crontab 临时失效，系统也会在用户/后台发起请求时做轻量到期检查（有节流），自动结束到期的开钓单，避免座位长期占用无法释放。
