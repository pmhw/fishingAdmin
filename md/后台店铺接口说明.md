# 后台 · 钓场店铺接口说明

前置：**执行** `md/venue_shop_product.sql` 建表，并执行其中 **权限 INSERT**（或为角色手动勾选 `admin.shop.product.manage`、`admin.shop.venue.manage`）。  
配置：`config/admin_permission.php` 已登记路由与权限码。

基础路径：`/api/admin`（与现有后台一致），需登录 + 对应权限。

---

## 一、公共商品库 `admin.shop.product.manage`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `shop/product-categories` | 分类列表（启用中） |
| GET | `shop/products` | SPU 分页；query: `page`,`limit`,`keyword`,`status`,`category_id` |
| GET | `shop/products/:id` | 详情 + 全部 SKU |
| POST | `shop/products` | 新建 SPU；body: `name` 必填，可选 `category_id`,`intro`,`cover_image`,`images`（数组或 JSON 字符串）,`detail`,`unit`,`status`,`sort_order` |
| PUT | `shop/products/:id` | 更新 SPU（传字段即更新） |
| DELETE | `shop/products/:id` | 删除 SPU（级联 SKU；若已被店铺引用会受外键/约束影响） |
| POST | `shop/products/:id/skus` | 新增规格；`spec_label` 必填；可选 `spec_json`,`sku_code`,`default_price`,`sort_order`,`status` |
| PUT | `shop/skus/:id` | 更新规格 |
| DELETE | `shop/skus/:id` | 删除规格 |

---

## 二、钓场店铺选品与库存 `admin.shop.venue.manage`

数据范围与 **`GET /api/admin/venue-options`** 一致：非超管且角色配置了 `admin_role_venue` 时，仅能操作被分配的 `venue_id`。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `shop/venues/:venue_id/products` | 本店已选商品分页；每条含 `product` 与 `skus`（含 `price`,`stock`,`status` 及 `product_sku`） |
| GET | `shop/venues/:venue_id/available-products` | 可选公共商品（`status=1` 且未加入本店）；query: `page`,`limit`,`keyword` |
| POST | `shop/venues/:venue_id/products` | 加入店铺；body: `product_id`；为所有**启用中**的公共 SKU 自动建行，价=`default_price` 或 `0`，库存 `0` |
| PUT | `shop/venues/:venue_id/products/:vp_id` | 本店 SPU 排序/上下架；body: `status`,`sort_order` |
| DELETE | `shop/venues/:venue_id/products/:vp_id` | 移除本店商品（级联删除本店 SKU 行） |
| PUT | `shop/venues/:venue_id/skus/batch` | 批量改售价/库存；body: `items: [{ id: venue_product_sku表主键, price?, stock?, status? }]` |
| POST | `shop/venues/:venue_id/products/:vp_id/sync` | 公共库新增规格后，为本店补建缺失的 `venue_product_sku` 行 |

说明：`vp_id` 为表 `venue_product` 的主键 `id`（非 `product_id`）。  
批量更新里每条 `id` 为表 `venue_product_sku` 的主键。

---

## 三、前端对接提示

1. 选品页：先 `available-products` 选 SPU，再 `POST .../products` 加入。  
2. 库存页：`GET .../products` 展开 `skus`，改价改库存后 `PUT .../skus/batch`。  
3. 公共库维护 SKU 后，已上架店铺可点「同步规格」调用 `POST .../sync`。

更完整的表结构说明见：`md/钓场店铺商品设计说明.md`。

## 四、本仓库内置管理界面（`admin/`）

使用 **Vue 3 + Element Plus**，与现有轮播图/钓场页面同一套请求封装。

| 路由路径 | 页面 | 所需权限 |
|----------|------|----------|
| `/shop/products` | 公共商品库：列表、SPU 编辑、规格 CRUD | `admin.shop.product.manage` |
| `/shop/venue` | 钓场选品：选钓场、从库添加、售价/库存批量保存、同步新规格 | `admin.shop.venue.manage` |

侧边栏 **「店铺管理」** 在拥有上述任一权限时显示；菜单项按权限细分。

前端 API 封装：`admin/src/api/shop.js`。  
部署后请在角色中勾选对应权限，并执行 `venue_shop_product.sql` 中的权限 INSERT（或手动添加权限码）。
