# 后台 · 钓场店铺接口说明

前置：**执行** `md/venue_shop_product.sql` 建表，并执行其中 **权限 INSERT**（或为角色手动勾选 `admin.shop.product.manage`、`admin.shop.venue.manage`）。  
配置：`config/admin_permission.php` 已登记路由与权限码。

基础路径：`/api/admin`（与现有后台一致），需登录 + 对应权限。

---

## 一、公共商品库 `admin.shop.product.manage`

**说明：** 公共商品**不选分类**；分类在各钓场「本店分类」维护（见第二节）。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `shop/products` | SPU 分页；query: `page`,`limit`,`keyword`,`status` |
| GET | `shop/products/:id` | 详情 + 全部 SKU |
| POST | `shop/products` | 新建 SPU；`name` 必填；可选 `intro`,`cover_image`,`images`,`detail`,`unit`,`status`,`sort_order` |
| PUT | `shop/products/:id` | 更新 SPU |
| DELETE | `shop/products/:id` | 删除 SPU |
| GET | `shop/product-skus` | 规格列表；**query 必填** `product_id`（与 `pond-return-rules?pond_id=` 用法类似） |
| POST | `shop/product-skus` | 新增规格；body: `product_id`,`spec_label` 必填；可选 `default_price`,`sort_order`,`status`,`spec_json`,`sku_code` |
| PUT | `shop/product-skus/:id` | 更新规格 |
| DELETE | `shop/product-skus/:id` | 删除规格 |

---

## 二、钓场店铺选品与库存 `admin.shop.venue.manage`

数据范围与 **`GET /api/admin/venue-options`** 一致：非超管且角色配置了 `admin_role_venue` 时，仅能操作被分配的 `venue_id`。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `shop/venues/:venue_id/categories` | **本店商品分类**列表（每钓场独立） |
| POST | `shop/venues/:venue_id/categories` | 新增分类；body: `name`,`sort_order?`,`status?` |
| PUT | `shop/venues/:venue_id/categories/:id` | 更新分类 |
| DELETE | `shop/venues/:venue_id/categories/:id` | 删除分类（`venue_product.shop_category_id` 会置空） |
| GET | `shop/venues/:venue_id/products` | 本店已选商品分页；query 可选 `shop_category_id`：`0` 或空串表示只看待分组；每条含 `shop_category`、`product`、`skus` |
| GET | `shop/venues/:venue_id/available-products` | 可选公共商品；query: `page`,`limit`,`keyword` |
| POST | `shop/venues/:venue_id/products` | 加入店铺；body: `product_id`；可选 `shop_category_id`（须为本店已启用分类） |
| PUT | `shop/venues/:venue_id/products/:vp_id` | 本店 SPU；body: `status`,`sort_order`，可选 `shop_category_id`（传空清除分类） |
| DELETE | `shop/venues/:venue_id/products/:vp_id` | 移除本店商品 |
| PUT | `shop/venues/:venue_id/skus/batch` | 批量改售价/库存 |
| POST | `shop/venues/:venue_id/products/:vp_id/sync` | 同步公共库新规格到本店 |

说明：`vp_id` 为表 `venue_product` 的主键 `id`（非 `product_id`）。  
批量更新里每条 `id` 为表 `venue_product_sku` 的主键。

## 三、前端对接提示

1. 公共规格：`GET/POST shop/product-skus`（带 `product_id`），交互可对齐池塘「回鱼规则」列表。  
2. 本店分类：先维护 `.../venues/:id/categories`，再在选品/设置里带 `shop_category_id`。  
3. 选品：`available-products` → `POST .../products`；库存：`GET .../products` + `PUT .../skus/batch`；同步新规格：`POST .../sync`。

更完整的表结构说明见：`md/钓场店铺商品设计说明.md`。

## 四、本仓库内置管理界面（`admin/`）

使用 **Vue 3 + Element Plus**，与现有轮播图/钓场页面同一套请求封装。

| 路由路径 | 页面 | 所需权限 |
|----------|------|----------|
| `/shop/products` | 公共商品库；**规格配置**为表内编辑+保存（同回鱼规则） | `admin.shop.product.manage` |
| `/shop/venue` | 钓场选品：**本店分类**、按分类筛选、加入/设置时可归组 | `admin.shop.venue.manage` |

侧边栏 **「店铺管理」** 在拥有上述任一权限时显示；菜单项按权限细分。

前端 API 封装：`admin/src/api/shop.js`。  
部署后请在角色中勾选对应权限，并执行 `venue_shop_product.sql` 中的权限 INSERT（或手动添加权限码）。
