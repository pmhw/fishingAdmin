# 支付说明（微信小程序 + 订单）

本文档说明本项目中微信支付与订单相关配置、接口及回调流程。

---

## 一、微信支付（小程序 JSAPI）

### 1. 配置

- **小程序与支付配置**（`.env`）：
  - `WECHAT_MINI_APPID`、`WECHAT_MINI_SECRET`（登录用）
  - `WECHAT_PAY_MCH_ID`、`WECHAT_PAY_KEY`、`WECHAT_PAY_NOTIFY_URL`（见 `config/wechat_pay.php`）

- **支付回调**  
  微信会 POST 到 `WECHAT_PAY_NOTIFY_URL`，由 `Api.Mini.PayController::notify()` 处理：验签、解密、根据 `out_trade_no` 更新订单状态（已支付等），并做幂等。

- **统一下单与调起支付**  
  - 接口：`POST /api/mini/pay/wechat/jsapi`（需登录）。  
  - Body：`order_no`、`description`、`total_fee`（分）。  
  - 返回 `wx.requestPayment` 所需参数；小程序用该参数调起 `wx.requestPayment`。  
  - 订单需先在本系统创建（`fishing_order` 等），`order_no` 与 `out_trade_no` 一致，回调里根据 `out_trade_no` 更新订单状态。

### 2. 路由

- `POST /api/mini/pay/wechat/jsapi` → `Api.Mini.PayController::jsapi`（需登录）
- `POST /api/mini/pay/wechat/notify` → `Api.Mini.PayController::notify`（微信回调，无需登录）

### 3. 小程序端

- 登录后拿到 token，请求 `/api/mini/pay/wechat/jsapi`，用返回参数调起 `wx.requestPayment`。
- 支付结果以服务端回调为准；前端可根据订单查询接口展示最终状态。

## 二、后台配置说明

### 1. 环境变量（.env）

最基础的支付配置在 `.env` 中设置：

- `WECHAT_MINI_APPID`：小程序 AppID  
- `WECHAT_MINI_SECRET`：小程序 AppSecret  
- `WECHAT_PAY_MCH_ID`：微信支付商户号（mch_id）  
- `WECHAT_PAY_KEY`：微信支付 API v2 密钥（32 位）  
- `WECHAT_PAY_NOTIFY_URL`：支付回调地址（完整 https URL，可不填，建议改为后台配置，见下）  

> 注意：`.env` 里的密钥信息只用于后端，不会返回给前端。

### 2. 配置文件（config/wechat_pay.php）

`config/wechat_pay.php` 会从 `.env` 读取上述变量，并提供默认回调地址：

- `mch_id`：`WECHAT_PAY_MCH_ID`  
- `key`：`WECHAT_PAY_KEY`  
- `notify_url`：`WECHAT_PAY_NOTIFY_URL` 或默认的 `https://你的域名/api/mini/pay/wechat/notify`  

通常只需保证 `.env` 中商户号、密钥设置正确即可。

### 3. 后台全局配置表（system_config）

为了在后台动态修改商户配置而不改代码，`PayController::jsapi()` 会优先从全局配置表读取部分字段：

- 表：`system_config`  
- 模型：`SystemConfig`  
- key：  
  - `pay_mch_id`：支付商户号（覆盖 `WECHAT_PAY_MCH_ID`）  
  - `pay_key`：支付 API v2 密钥（覆盖 `WECHAT_PAY_KEY`）  
  - `pay_notify_url`：支付回调地址（覆盖 `WECHAT_PAY_NOTIFY_URL`）

读取顺序：

1. 先读 `SystemConfig::getValue('pay_mch_id', '')`，若有值则用作 `mch_id`，否则用 `WECHAT_PAY_MCH_ID`；  
2. 再读 `SystemConfig::getValue('pay_key', '')`，若有值则用作 API v2 密钥，否则用 `WECHAT_PAY_KEY`；  
3. 最后读 `SystemConfig::getValue('pay_notify_url', '')`，若有值则用作 `notify_url`，否则用 `config/wechat_pay.php` 中的 `notify_url`（`.env`）。

#### 后台配置步骤

1. 登录后台管理 → 打开「系统配置」页面（`/api/admin/configs` 对应的管理界面）。  
2. 点击「新增」配置，可按需添加：  
   - `config_key`：`pay_mch_id`，`config_value`：你的微信支付商户号，`remark`：如「微信支付商户号」  
   - `config_key`：`pay_key`，`config_value`：你的 API v2 密钥（32 位），`remark`：如「微信支付 APIv2 密钥」  
   - `config_key`：`pay_notify_url`，`config_value`：`https://你的真实域名/api/mini/pay/wechat/notify`，`remark`：如「微信小程序支付回调地址」  
3. 保存后，无需重启服务，新的配置会在后端统一下单时自动生效。

> 如果同时配置了 `.env` 和后台上述 key（如 `pay_mch_id`、`pay_key`、`pay_notify_url`），**以后台配置为准**。

## 三、订单与回调（简要）

- 订单表（如 `fishing_order`）存 `order_no`（= 微信 `out_trade_no`）、金额、状态等。
- 支付回调（`notify`）：验签 → 解密 → 用 `out_trade_no` 查订单 → 更新为已支付并写 `transaction_id` 等，幂等处理重复回调。
